## О проекте

**Marketplace** — система для агрегации и анализа данных с маркетплейсов с умными рекомендациями, персистентной корзиной и беспарольной авторизацией через биометрию.

### Функциональность:
- Агрегация товаров с различных маркетплейсов
- Парсинг цен и характеристик товаров
- Динамическое изменение цены
- Категоризация товаров
- Поиск и фильтрация
- Passkeys / WebAuthn — беспарольная авторизация через биометрию или PIN устройства
- Персистентная корзина — синхронизируется с БД при входе, переживает перезагрузку страницы
- Система рекомендаций — блок «Вам может понравиться» на главной странице
- Сохранение заказов — история заказов привязана к аккаунту пользователя
- Сбор событий — просмотры, добавления в корзину и покупки учитываются при персонализации

## Авторизация — Passkeys / WebAuthn

Вместо паролей используются криптографические ключи, хранящиеся на устройстве (TPM, Secure Enclave, менеджер паролей). Приватный ключ **никогда не покидает устройство**.

- Одна учётная запись поддерживает несколько устройств — каждое регистрирует свой ключ
- Сессия хранится как 64-байтный Bearer-токен (30 дней)
- При входе токен и корзина восстанавливаются из localStorage через Redux Persist

Подробнее: [AUTH.md](../AUTH.md)

## Система рекомендаций

Блок «Вам может понравиться» на главной странице работает по трём стратегиям:

| Стратегия | Когда применяется | Как работает |
|-----------|------------------|-------------|
| **Content-based** | Всегда (холодный старт) | Cosine similarity по атрибутам товара (категория, цвет, материал, цена) |
| **Cart-based** | Есть товары в корзине | Агрегация content-рекомендаций для всех товаров в корзине |
| **Collaborative** | Авторизован + 50+ событий | TF Recommenders two-tower модель по истории взаимодействий |

Обучение запускается вручную:
```bash
python manage.py train_recommender --strategy content      # сразу после деплоя
python manage.py train_recommender --strategy all          # после накопления событий
```
